# ユーザードキュメントアクセス機能 要件定義書

## 1. 概要

### 1.1 目的
システム内の各ユーザーページから、ロールに応じた利用マニュアルに簡単にアクセスできるようにする。

### 1.2 背景
- 現在、`doc/manual/` にユーザーマニュアルが整備されている（参加者、主催者、審査員、管理者向け）
- しかし、アプリケーション内からこれらのドキュメントにアクセスする手段がない
- ユーザーが困った時に自己解決できる仕組みが必要

### 1.3 対象ユーザー
| ロール | 対応マニュアル |
|--------|---------------|
| 参加者（participant） | participant_guide.md |
| 主催者（organizer） | organizer_guide.md |
| 審査員（judge） | judge_guide.md |
| 管理者（admin） | admin_guide.md |

## 2. 機能要件

### 2.1 ヘルプページの作成

#### FR-001: マニュアル一覧ページ
- **説明**: 全マニュアルの一覧を表示するページを作成する
- **URL**: `/help`
- **内容**:
  - 4つのマニュアルへのリンク（カード形式）
  - 各マニュアルの概要説明
  - ユーザーのロールに応じて推奨マニュアルをハイライト
- **アクセス権限**: 全ユーザー（未ログインでも閲覧可能）

#### FR-002: 個別マニュアルページ
- **説明**: 各マニュアルを表示するページを作成する
- **URL**:
  - `/help/participant` - 参加者向けマニュアル
  - `/help/organizer` - 主催者向けマニュアル
  - `/help/judge` - 審査員向けマニュアル
  - `/help/admin` - 管理者向けマニュアル
- **内容**:
  - Markdownファイルをパースして表示
  - 目次（アンカーリンク付き）
  - 前後のセクションへのナビゲーション
  - マニュアル一覧へ戻るリンク
- **アクセス権限**: 全ユーザー（未ログインでも閲覧可能）

### 2.2 ナビゲーションへのリンク追加

#### FR-003: ヘッダーナビゲーション
- **説明**: メインヘッダーにヘルプリンクを追加する
- **配置**: ユーザーメニュー（プロフィールドロップダウン）内
- **表示テキスト**: 「ヘルプ」または「利用ガイド」
- **リンク先**: `/help`（一覧ページ）
- **対象**: 全ユーザー（ログイン・未ログイン問わず）

#### FR-004: フッターリンク
- **説明**: 既存フッターのプレースホルダーリンクを有効化
- **追加項目**: 「利用ガイド」→ `/help`
- **対象**: 全ページ共通

#### FR-005: 主催者サイドバー
- **説明**: 主催者用サイドバーにヘルプセクションを追加
- **配置**: サイドバー最下部
- **内容**:
  - セクション名: 「ヘルプ」
  - リンク: 「主催者ガイド」→ `/help/organizer`
  - リンク: 「全マニュアル」→ `/help`

#### FR-006: 管理者ナビゲーション
- **説明**: 管理画面のナビゲーションにヘルプリンクを追加
- **配置**: 管理画面ヘッダーの右側
- **リンク**: 「管理者ガイド」→ `/help/admin`

#### FR-007: 審査員ダッシュボード
- **説明**: 審査員ダッシュボードにヘルプリンクを追加
- **配置**: ダッシュボードページ上部またはサイドパネル
- **リンク**: 「審査員ガイド」→ `/help/judge`
- **表示条件**: 審査員として割り当てられているユーザーのみ

### 2.3 コンテキストヘルプ

#### FR-008: ページ別ヘルプリンク
- **説明**: 特定のページに関連するマニュアルセクションへの直接リンク
- **対象ページと対応セクション**:

| ページ | リンク先 |
|--------|---------|
| コンテスト応募フォーム | `/help/participant#3-写真を応募する` |
| コンテスト作成フォーム | `/help/organizer#3-コンテストの作成` |
| モデレーション画面 | `/help/organizer#6-コンテンツモデレーション` |
| 評価フォーム | `/help/judge#4-作品の評価` |
| 審査員招待受諾ページ | `/help/judge#2-招待の受諾` |

- **表示形式**: 「?」アイコン または 「このページのヘルプ」リンク

## 3. 非機能要件

### 3.1 パフォーマンス

#### NFR-001: ページ読み込み速度
- マニュアルページは1秒以内に表示されること
- Markdownのパースはサーバーサイドで行い、キャッシュを活用する

#### NFR-002: キャッシュ
- パースされたHTMLをキャッシュする（変更検知でキャッシュ更新）
- キャッシュキー: ファイルの更新日時を含める

### 3.2 アクセシビリティ

#### NFR-003: レスポンシブデザイン
- モバイル、タブレット、デスクトップで適切に表示されること
- 目次はモバイルでは折りたたみ式にする

#### NFR-004: 見出し構造
- 適切な見出しレベル（h1, h2, h3）を使用する
- スクリーンリーダーで読み上げ可能な構造にする

### 3.3 保守性

#### NFR-005: Markdownファイルの管理
- マニュアルは引き続き `doc/manual/` でMarkdownファイルとして管理
- アプリケーションコードとドキュメントを分離
- Markdownファイルの更新がそのまま反映される仕組み

#### NFR-006: 国際化対応（将来）
- 将来的に多言語対応できる構造にする
- 現時点では日本語のみ

## 5. 技術設計

### 5.1 ルーティング

```ruby
# config/routes.rb
get 'help', to: 'help#index'
get 'help/:guide', to: 'help#show', as: :help_guide,
    constraints: { guide: /participant|organizer|judge|admin/ }
```

### 5.2 コントローラー

```ruby
# app/controllers/help_controller.rb
class HelpController < ApplicationController
  # ログイン不要でアクセス可能
  skip_before_action :authenticate_user!

  def index
    # マニュアル一覧を表示
  end

  def show
    @guide = params[:guide]
    @content = render_markdown("doc/manual/#{@guide}_guide.md")
  end

  private

  def render_markdown(path)
    # Markdownをパースしてキャッシュ
    Rails.cache.fetch(cache_key(path)) do
      Redcarpet::Markdown.new(renderer, options).render(File.read(path))
    end
  end
end
```

### 5.3 必要なGem

```ruby
# Gemfile
gem 'redcarpet'  # Markdownパーサー
gem 'rouge'      # シンタックスハイライト（コードブロック用）
```

### 5.4 ビュー

```
app/views/help/
├── index.html.erb    # マニュアル一覧
├── show.html.erb     # 個別マニュアル表示
└── _toc.html.erb     # 目次パーシャル
```

## 6. 受け入れ条件

### AC-001: マニュアル一覧ページ
- [ ] `/help` にアクセスすると4つのマニュアルカードが表示される
- [ ] ログインしていなくてもアクセスできる
- [ ] 各カードをクリックすると対応するマニュアルページに遷移する

### AC-002: 個別マニュアルページ
- [ ] `/help/participant` で参加者向けマニュアルが表示される
- [ ] `/help/organizer` で主催者向けマニュアルが表示される
- [ ] `/help/judge` で審査員向けマニュアルが表示される
- [ ] `/help/admin` で管理者向けマニュアルが表示される
- [ ] 目次が表示され、クリックで該当セクションにスクロールする
- [ ] Markdownの書式（見出し、表、リスト、コードブロック）が正しく表示される

### AC-003: ナビゲーションリンク
- [ ] ヘッダーのユーザーメニューに「ヘルプ」リンクがある
- [ ] フッターに「利用ガイド」リンクがある
- [ ] 主催者サイドバーに「ヘルプ」セクションがある
- [ ] 管理画面ヘッダーに「管理者ガイド」リンクがある
- [ ] 審査員ダッシュボードに「審査員ガイド」リンクがある

### AC-004: レスポンシブ対応
- [ ] モバイルでマニュアル一覧が1カラムで表示される
- [ ] モバイルで目次が折りたたみ式になる
- [ ] 全画面サイズで読みやすいフォントサイズになっている

## 8. 備考

### 既存システムとの整合性
- 既存のチュートリアルシステム（オンボーディング用）とは独立した機能として実装
- チュートリアルは「操作を学ぶ」、マニュアルは「参照する」という役割分担

### 更新フロー
- マニュアルの更新は `doc/manual/` のMarkdownファイルを編集
- デプロイ時にキャッシュがクリアされ、最新版が反映される
