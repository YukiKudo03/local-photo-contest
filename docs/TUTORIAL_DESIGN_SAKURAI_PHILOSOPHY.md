# チュートリアルデザイン設計書
## 桜井政博氏のゲームデザイン哲学に基づく設計

---

## 1. 設計思想の背景

### 1.1 桜井政博氏について

桜井政博氏は『星のカービィ』『大乱闘スマッシュブラザーズ』シリーズの生みの親であり、「初心者と上級者の同居」という設計思想で知られています。YouTubeチャンネル「桜井政博のゲーム作るには」では、ゲーム設計のノウハウを公開し、CEDEC AWARDS 2023でゲームデザイン部門最優秀賞を受賞しています。

### 1.2 本設計書の目的

Local Photo Contestのチュートリアルシステムに、桜井氏の設計哲学を取り入れ、以下を実現します：

- ユーザーが「やりたいこと」にすぐ到達できる体験
- 説明を読ませるのではなく、体験を通じて学ばせる設計
- 初心者から熟練者まで、すべてのユーザーが快適に使えるシステム

## 2. 核となる設計原則

### 2.1 「とにかくゲームさせてみて」の原則

> *"物事を入れ替えて、本題（面白い部分）から見せる"*
> — 桜井政博

#### 適用方針

**従来のアプローチ（避けるべき）:**
```
1. 長い説明文を表示
2. 機能の概要を説明
3. 注意事項を列挙
4. ようやく操作開始
```

**桜井流アプローチ（採用）:**
```
1. すぐに操作させる
2. 操作中に最小限のヒントを表示
3. 成功体験を与える
4. 詳細は後から（必要な人だけに）
```

#### 具体的実装

| 機能 | 従来型 | 桜井流 |
|------|--------|--------|
| 写真投稿 | 投稿フォームの説明→各フィールドの解説→投稿 | 写真をドロップ→即プレビュー→必要項目だけ入力→完了 |
| 投票 | 投票ルールの説明→投票方法の解説→投票 | 作品一覧を表示→気になる作品をタップ→ハートで投票→完了 |
| コンテスト作成 | 全フィールドの説明→テンプレート説明→作成 | テンプレート選択→タイトル入力→すぐプレビュー→公開 |

### 2.2 「初心者と上級者の同居」の原則

> *"ゲーム初心者に優しい作りでありながら上級者でも楽しめるように"*
> — 桜井政博（カービィ開発時の設計思想）

#### 二層構造の設計

```
┌─────────────────────────────────────────────────────────┐
│                    上級者向けレイヤー                      │
│  ・詳細設定へのクイックアクセス                            │
│  ・ショートカット操作                                     │
│  ・高度な分析機能                                         │
│  ・カスタマイズオプション                                  │
├─────────────────────────────────────────────────────────┤
│                    初心者向けレイヤー                      │
│  ・シンプルな操作フロー                                   │
│  ・デフォルト値の最適化                                   │
│  ・段階的な機能開放                                       │
│  ・コンテキストヘルプ                                     │
└─────────────────────────────────────────────────────────┘
```

#### 具体的実装

**写真投稿（初心者モード）:**
- 必須項目のみ表示（写真、タイトル）
- 「もっと詳しく」ボタンで追加項目を展開
- 推奨設定が自動適用

**写真投稿（上級者モード）:**
- すべての項目が最初から表示
- EXIF情報の自動読み取り
- 過去の設定を記憶

### 2.3 「待たせない」の原則

> *"操作できない時間はストレス。足りなかったのは説明ではなく、テンポだった。"*
> — ソーシャルゲーム開発者の実例より

#### 時間制約の設計指針

| 要素 | 最大時間 | 理由 |
|------|---------|------|
| ウェルカムモーダル | 5秒で閉じられる | 第一印象で離脱を防ぐ |
| 各ステップの説明文 | 3秒で読める量 | 集中力の維持 |
| アニメーション | 0.3秒以下 | テンポ感の確保 |
| 次のアクションまで | 即座に可能 | ストレス軽減 |

#### 文字数制限

```ruby
# チュートリアルステップの文字数制限
TUTORIAL_TEXT_LIMITS = {
  title: 20,        # タイトル：20文字以内
  description: 60,  # 説明文：60文字以内（3秒で読める）
  hint: 30          # ヒント：30文字以内
}.freeze
```

### 2.4 「失敗させない」の原則

> *"カービィは空を飛べるので、落下死しにくい。初心者が安心して遊べる設計。"*
> — カービィのゲームデザイン

#### セーフティネットの設計

```
┌─────────────────────────────────────────────┐
│              操作のセーフティネット            │
├─────────────────────────────────────────────┤
│                                             │
│  [写真投稿]                                  │
│   ├─ 自動保存（下書き）                       │
│   ├─ 投稿前プレビュー                         │
│   ├─ 「戻る」で入力内容保持                    │
│   └─ 投稿後も編集可能                         │
│                                             │
│  [コンテスト作成]                             │
│   ├─ テンプレートからの開始                    │
│   ├─ 段階的な入力（最小構成から）               │
│   ├─ 公開前の確認画面                         │
│   └─ 下書き保存機能                           │
│                                             │
│  [投票]                                      │
│   ├─ 投票取り消し可能                         │
│   ├─ 制限の事前表示                           │
│   └─ 確認なしの即時反映（スピード優先）         │
│                                             │
└─────────────────────────────────────────────┘
```

### 2.5 「褒める・フィードバック」の原則

> *"成功したらすぐにフィードバック"*
> — ゲームデザインの基本原則

#### フィードバックの設計

| アクション | 即時フィードバック | 詳細フィードバック |
|-----------|-------------------|-------------------|
| 写真投稿完了 | 成功アニメーション + 効果音 | 「素敵な作品ですね！」メッセージ |
| 初投票 | ハートのアニメーション | 「初めての投票おめでとう！」バッジ |
| コンテスト公開 | 紙吹雪エフェクト | 共有ボタン + 次のステップ案内 |
| 審査完了 | チェックマーク | 進捗バーの更新 + 称賛メッセージ |

## 5. 実装ガイドライン

### 5.1 チュートリアルの構造

```ruby
# 桜井流チュートリアルの実装パターン
class TutorialStep
  # 1ステップあたりの制限
  MAX_TITLE_LENGTH = 20
  MAX_DESCRIPTION_LENGTH = 60
  MAX_DURATION_SECONDS = 10

  # 必須要素
  validates :title, length: { maximum: MAX_TITLE_LENGTH }
  validates :description, length: { maximum: MAX_DESCRIPTION_LENGTH }

  # 動詞で始まるタイトルを推奨
  validate :title_starts_with_action_verb

  private

  def title_starts_with_action_verb
    action_verbs = %w[クリック タップ 入力 選択 確認 見て]
    unless action_verbs.any? { |verb| title.start_with?(verb) }
      # 警告のみ（エラーにはしない）
      Rails.logger.warn "Tutorial title should start with action verb: #{title}"
    end
  end
end
```

### 5.2 フィードバックの実装

```javascript
// 成功フィードバックコントローラー
export default class extends Controller {
  static values = {
    message: { type: String, default: "完了しました！" },
    type: { type: String, default: "success" }
  }

  // 即座にフィードバックを表示（桜井流）
  trigger() {
    // 1. 視覚的フィードバック（即座）
    this.showVisualFeedback()

    // 2. 音声フィードバック（オプション）
    this.playSound()

    // 3. メッセージ表示（0.2秒後）
    setTimeout(() => this.showMessage(), 200)

    // 4. 次のアクションを提示（1秒後）
    setTimeout(() => this.suggestNextAction(), 1000)
  }

  showVisualFeedback() {
    this.element.classList.add('feedback-success')
    // アニメーション終了後にクラスを削除
    setTimeout(() => {
      this.element.classList.remove('feedback-success')
    }, 300)
  }
}
```

## 6. 測定と改善

### 6.1 重要指標（KPI）

| 指標 | 目標値 | 測定方法 |
|------|--------|---------|
| チュートリアル完了率 | > 80% | TutorialProgress集計 |
| チュートリアル完了時間 | < 2分 | started_at〜completed_at |
| スキップ率 | < 20% | skipped件数 / 開始件数 |
| 初回アクション完了率 | > 90% | 初回ログイン後24時間以内 |
| 離脱ポイント | 各ステップで5%以下 | ステップ別離脱分析 |
