# Requirements Document

## Introduction

写真投稿機能は、参加者がフォトコンテストに写真を応募できる機能を提供する。写真のアップロード、タイトル・説明の入力、撮影場所情報の登録、応募状況の確認ができる。公開中のコンテストに対してのみ応募可能で、応募期間内であることを検証する。

## Alignment with Product Vision

この機能は product.md で定義された「写真投稿: 参加者が写真と撮影場所情報をアップロードできる応募機能」を実現する。応募完了率の成功指標（5分以内で完了）に貢献する。

---

## Requirements

### Requirement 1: コンテスト応募一覧表示

**User Story:** 参加者として、応募可能なコンテストの一覧を確認したい。それにより、参加したいコンテストを見つけられる。

#### Acceptance Criteria

1. WHEN 参加者がコンテスト一覧ページにアクセスする THEN システムは公開中で応募受付中のコンテスト一覧を表示する
2. WHEN コンテスト一覧を表示する THEN システムはコンテスト名、テーマ、応募期間、サムネイルを表示する
3. IF 応募受付中のコンテストが存在しない THEN システムは「現在応募可能なコンテストはありません」というメッセージを表示する
4. WHEN 参加者がコンテストカードをクリックする THEN システムはコンテスト詳細ページに遷移する

---

### Requirement 2: コンテスト詳細表示（参加者向け）

**User Story:** 参加者として、コンテストの詳細情報を確認したい。それにより、応募前にコンテストの内容を理解できる。

#### Acceptance Criteria

1. WHEN 参加者がコンテスト詳細ページにアクセスする THEN システムはコンテストのタイトル、説明、テーマ、応募期間を表示する
2. WHEN コンテストが応募受付中 THEN システムは「応募する」ボタンを表示する
3. IF コンテストが応募期間外 THEN システムは応募ボタンを非表示または無効化し、「応募期間外です」と表示する
4. IF コンテストが終了済み THEN システムは「このコンテストは終了しました」と表示する

---

### Requirement 3: 写真投稿（応募）

**User Story:** 参加者として、コンテストに写真を応募したい。それにより、自分の作品でコンテストに参加できる。

#### Acceptance Criteria

1. WHEN 参加者が「応募する」ボタンをクリックする THEN システムは写真投稿フォームを表示する
2. WHEN 参加者がフォームに必要情報を入力して送信する THEN システムは応募を保存する
3. IF 必須項目（写真）が未入力 THEN システムはエラーメッセージを表示して保存しない
4. IF 写真ファイルサイズが10MBを超える THEN システムはエラーメッセージを表示する
5. WHEN 応募が完了する THEN システムは確認画面を表示し、応募完了メッセージを表示する
6. IF 同一ユーザーが同一コンテストに既に応募済み THEN システムは追加応募として処理する（1コンテスト複数応募可）

**必須入力項目:**
- 写真ファイル（JPEG, PNG, GIF、10MB以下）

**任意入力項目:**
- タイトル（100文字以内）
- 説明文（テキスト）
- 撮影場所（テキスト）
- 撮影日

---

### Requirement 4: 自分の応募一覧表示

**User Story:** 参加者として、自分が応募した作品の一覧を確認したい。それにより、応募状況を把握できる。

#### Acceptance Criteria

1. WHEN 参加者がマイページにアクセスする THEN システムは自分の応募作品一覧を表示する
2. WHEN 応募一覧を表示する THEN システムは写真サムネイル、タイトル、応募したコンテスト名、応募日時を表示する
3. IF 応募がない THEN システムは「応募作品がありません」というメッセージを表示する
4. WHEN 参加者が応募作品をクリックする THEN システムは応募詳細画面を表示する

---

### Requirement 5: 応募詳細表示

**User Story:** 参加者として、自分の応募作品の詳細を確認したい。それにより、応募内容を確認できる。

#### Acceptance Criteria

1. WHEN 参加者が応募詳細ページにアクセスする THEN システムは応募した写真、タイトル、説明、撮影情報を表示する
2. WHEN 応募詳細を表示する THEN システムは応募したコンテスト情報も表示する
3. IF コンテストが応募受付中 AND 応募者本人 THEN システムは編集・削除ボタンを表示する

---

### Requirement 6: 応募編集

**User Story:** 参加者として、自分の応募内容を編集したい。それにより、誤りを修正できる。

#### Acceptance Criteria

1. WHEN 参加者が編集ボタンをクリックする THEN システムは編集フォームを表示する
2. WHEN 参加者が変更を保存する THEN システムは更新された情報を保存する
3. IF 他のユーザーの応募を編集しようとする THEN システムはアクセスを拒否する
4. IF コンテストの応募期間が終了している THEN システムは編集を拒否する

---

### Requirement 7: 応募削除

**User Story:** 参加者として、自分の応募を取り消したい。それにより、応募をやめることができる。

#### Acceptance Criteria

1. WHEN 参加者が削除ボタンをクリックする THEN システムは確認ダイアログを表示する
2. WHEN 参加者が削除を確認する THEN システムは応募を削除する
3. IF 他のユーザーの応募を削除しようとする THEN システムはアクセスを拒否する
4. IF コンテストの応募期間が終了している THEN システムは削除を拒否する

---

### Requirement 8: 運営者による応募管理

**User Story:** 運営者として、コンテストへの応募を確認したい。それにより、応募状況を把握できる。

#### Acceptance Criteria

1. WHEN 運営者がコンテスト詳細画面で応募一覧タブをクリックする THEN システムは応募一覧を表示する
2. WHEN 応募一覧を表示する THEN システムは写真サムネイル、タイトル、応募者名、応募日時を表示する
3. IF 応募がない THEN システムは「応募がありません」というメッセージを表示する
4. WHEN 運営者が応募をクリックする THEN システムは応募詳細を表示する

---

## Non-Functional Requirements

### Code Architecture and Modularity
- **Single Responsibility Principle**: Entry モデルは応募情報のみを管理
- **Modular Design**: EntriesController は標準的な RESTful パターンに従う
- **Dependency Management**: 既存の Contest モデルに依存
- **Clear Interfaces**: Active Storage を使用した画像管理

### Performance
- 画像アップロード: 10MB以下
- 画像処理: サムネイル自動生成
- 一覧表示: 1秒以内

### Security
- 参加者認証が必要
- 応募者本人のみ編集・削除可能
- 運営者は自分のコンテストの応募のみ閲覧可能
- CSRF対策（Rails標準）

### Reliability
- 画像は Active Storage で管理
- アップロード失敗時は適切なエラーメッセージ

### Usability
- フォームは日本語ラベルを使用
- エラーメッセージは日本語で表示
- ドラッグ＆ドロップでの画像アップロード対応
