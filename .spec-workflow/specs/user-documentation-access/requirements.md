# Requirements: User Documentation Access

## Overview

システム内の各ユーザーページから、ロールに応じた利用マニュアルに簡単にアクセスできるようにする機能。

## Background

- `doc/manual/` にユーザーマニュアルが整備されている（参加者、主催者、審査員、管理者向け）
- アプリケーション内からこれらのドキュメントにアクセスする手段がない
- ユーザーが困った時に自己解決できる仕組みが必要

## User Stories

### US-001: 一般ユーザーとしてマニュアル一覧を見たい

**As a** 一般ユーザー
**I want to** マニュアル一覧ページにアクセスしたい
**So that** 自分に適したマニュアルを見つけられる

**Acceptance Criteria:**

- `/help` にアクセスすると4つのマニュアル（参加者、主催者、審査員、管理者）がカード形式で表示される
- 各カードにはタイトル、概要説明、アイコンが表示される
- ログインしていなくてもアクセスできる
- カードをクリックすると対応するマニュアルページに遷移する

### US-002: ユーザーとして個別マニュアルを読みたい

**As a** ユーザー
**I want to** 個別のマニュアルページを閲覧したい
**So that** 操作方法や機能について詳しく知ることができる

**Acceptance Criteria:**

- `/help/participant` で参加者向けマニュアルが表示される
- `/help/organizer` で主催者向けマニュアルが表示される
- `/help/judge` で審査員向けマニュアルが表示される
- `/help/admin` で管理者向けマニュアルが表示される
- 目次が表示され、クリックで該当セクションにスクロールする
- Markdownの書式（見出し、表、リスト、コードブロック）が正しく表示される
- 「マニュアル一覧に戻る」リンクがある

### US-003: ユーザーとしてヘッダーからヘルプにアクセスしたい

**As a** ユーザー
**I want to** ヘッダーナビゲーションからヘルプにアクセスしたい
**So that** どのページからでも簡単にマニュアルを参照できる

**Acceptance Criteria:**

- ユーザーメニュー（プロフィールドロップダウン）内に「ヘルプ」リンクがある
- モバイルメニューにも「ヘルプ」リンクがある
- クリックすると `/help` に遷移する
- ログイン・未ログイン問わず表示される

### US-004: ユーザーとしてフッターからヘルプにアクセスしたい

**As a** ユーザー
**I want to** フッターからヘルプにアクセスしたい
**So that** ページ下部からもマニュアルを参照できる

**Acceptance Criteria:**

- フッターに「利用ガイド」リンクがある
- クリックすると `/help` に遷移する

### US-005: 主催者としてサイドバーからガイドにアクセスしたい

**As a** 主催者
**I want to** サイドバーから主催者ガイドに直接アクセスしたい
**So that** コンテスト運営中に素早くヘルプを参照できる

**Acceptance Criteria:**

- 主催者用サイドバーの最下部に「ヘルプ」セクションがある
- 「主催者ガイド」リンクが `/help/organizer` に遷移する
- 「全マニュアル」リンクが `/help` に遷移する

### US-006: 管理者として管理画面からガイドにアクセスしたい

**As a** 管理者
**I want to** 管理画面から管理者ガイドに直接アクセスしたい
**So that** システム管理中に素早くヘルプを参照できる

**Acceptance Criteria:**

- 管理画面ヘッダーに「管理者ガイド」リンクがある
- クリックすると `/help/admin` に遷移する

### US-007: 審査員としてダッシュボードからガイドにアクセスしたい

**As a** 審査員
**I want to** 審査員ダッシュボードから審査員ガイドにアクセスしたい
**So that** 評価方法について素早く確認できる

**Acceptance Criteria:**

- 審査員ダッシュボード（`/my/judge_assignments`）に「審査員ガイド」へのリンクがある
- クリックすると `/help/judge` に遷移する

## Non-Functional Requirements

### NFR-001: パフォーマンス

- マニュアルページは1秒以内に表示されること
- Markdownのパースはサーバーサイドで行い、キャッシュを活用する

### NFR-002: キャッシュ

- パースされたHTMLをキャッシュする
- ファイルの更新日時をキャッシュキーに含め、変更検知でキャッシュ更新

### NFR-003: レスポンシブデザイン

- モバイル、タブレット、デスクトップで適切に表示されること
- 目次はモバイルでは折りたたみ式にする
- カード一覧はモバイルで1カラム表示

### NFR-004: アクセシビリティ

- 適切な見出しレベル（h1, h2, h3）を使用する
- スクリーンリーダーで読み上げ可能な構造にする
- リンクは明確なラベルを持つ

### NFR-005: 保守性

- マニュアルは引き続き `doc/manual/` でMarkdownファイルとして管理
- アプリケーションコードとドキュメントを分離
- Markdownファイルの更新がそのまま反映される仕組み

### NFR-006: 国際化対応

- 現時点では日本語のみ
- 将来的に多言語対応できる構造にする（i18n使用）

## Constraints

- 新規データベースモデルは不要
- 既存のチュートリアルシステムとは独立した機能として実装
- 認証不要でアクセス可能（SEO対応）

## Out of Scope

- コンテキストヘルプ（ページ別の直接リンク）は Phase 3 で検討
- マニュアルの編集機能（管理画面から）
- マニュアルの検索機能
- PDF出力機能

## Dependencies

- `doc/manual/*.md` ファイルが存在すること
- Redcarpet gem（Markdownパーサー）
- Rouge gem（シンタックスハイライト）
