# Requirements Document: Area Management (地域指定機能)

## Introduction

商店街単位でのフォトコンテスト開催を支援するための地域指定機能。主催者が対象エリアを定義し、撮影スポット（店舗・名所）を登録することで、地域に密着したコンテスト運営を可能にする。地図上でのエリア境界表示やスポットマーカー表示により、参加者が撮影場所を視覚的に把握できる。

## Alignment with Product Vision

本機能は product.md に記載された以下の目標に直接貢献する：

- **地域密着**: 商店街・町内会など小規模エリアでのコンテスト開催を支援
- **地域限定機能**: 撮影場所の指定、地域タグ付け、エリア別カテゴリ分け
- **ギャラリー表示**: 応募作品を地域マップや撮影スポット別に閲覧
- **地域カバー率**: 対象地域内の主要スポットからの応募を促進

## Requirements

### REQ-1: エリア作成

**User Story:** As a コンテスト主催者, I want エリア（地域）を作成できる, so that コンテストの対象地域を明確に定義できる

#### Acceptance Criteria

1. WHEN 主催者がエリア作成画面にアクセス THEN システムは新規エリア作成フォームを表示する
2. WHEN 主催者がエリア名、住所情報を入力して保存 THEN システムはエリアを作成し一覧に追加する
3. IF エリア名が空 THEN システムは「エリア名を入力してください」エラーを表示する
4. IF 同一主催者が同名のエリアを作成しようとした THEN システムは「同じ名前のエリアが既に存在します」エラーを表示する
5. WHEN 住所を入力 THEN システムは都道府県・市区町村・詳細住所を個別フィールドで管理する

### REQ-2: エリア地図設定

**User Story:** As a コンテスト主催者, I want 地図上でエリアの代表位置と境界線を設定できる, so that 参加者が対象エリアを視覚的に把握できる

#### Acceptance Criteria

1. WHEN エリア編集画面を開く THEN システムは Leaflet + OpenStreetMap の地図を表示する
2. WHEN 住所が入力済み THEN システムは住所から座標を取得し地図の中心に設定する
3. WHEN 主催者が地図上をクリック THEN システムは代表座標（緯度・経度）を設定する
4. WHEN 主催者が境界線描画モードを有効にして地図上をクリック THEN システムはポリゴンの頂点を追加する
5. WHEN 主催者が境界線を閉じる THEN システムは GeoJSON 形式で境界データを保存する
6. IF 境界線が未設定 THEN システムは代表座標のみを保存し、境界線なしでも動作する

### REQ-3: エリア一覧・管理

**User Story:** As a コンテスト主催者, I want 作成したエリアを一覧表示・編集・削除できる, so that エリア情報を適切に管理できる

#### Acceptance Criteria

1. WHEN 主催者がエリア管理画面にアクセス THEN システムは主催者が作成したエリア一覧を表示する
2. WHEN 主催者がエリアを選択 THEN システムは詳細情報と地図プレビューを表示する
3. WHEN 主催者がエリアを編集 THEN システムはエリア情報を更新する
4. WHEN 主催者がエリアを削除 AND 紐づくコンテストがない THEN システムはエリアを削除する
5. IF エリアに紐づくコンテストが存在 THEN システムは「このエリアはコンテストで使用中のため削除できません」と表示する

### REQ-4: スポット作成

**User Story:** As a コンテスト主催者, I want コンテストに撮影スポット（店舗・名所）を登録できる, so that 参加者が撮影対象を把握できる

#### Acceptance Criteria

1. WHEN 主催者がコンテストのスポット管理画面にアクセス THEN システムはスポット一覧と追加フォームを表示する
2. WHEN 主催者がスポット名とカテゴリを入力して保存 THEN システムはスポットを作成しコンテストに紐付ける
3. IF スポット名が空 THEN システムは「スポット名を入力してください」エラーを表示する
4. WHEN 主催者が地図上をクリックしてスポットを追加 THEN システムはクリック位置の座標でスポットを作成する
5. WHEN スポットにカテゴリを設定 THEN システムは以下のカテゴリから選択できる：飲食店、小売店、サービス業、名所・ランドマーク、公共施設、公園・広場、寺社仏閣、その他

### REQ-5: スポット管理

**User Story:** As a コンテスト主催者, I want スポットを編集・削除・並び替えできる, so that 撮影スポット情報を最新に保てる

#### Acceptance Criteria

1. WHEN 主催者がスポットを選択 THEN システムは編集フォームを表示する
2. WHEN 主催者がスポット情報を更新して保存 THEN システムはスポット情報を更新する
3. WHEN 主催者がスポットを削除 AND 紐づく投稿がない THEN システムはスポットを削除する
4. IF スポットに紐づく投稿が存在 THEN システムは「このスポットは投稿で使用中です。削除すると投稿のスポット情報がクリアされます。続行しますか？」と確認する
5. WHEN 主催者がスポットの表示順を変更 THEN システムは position を更新し一覧の順序を反映する

### REQ-6: コンテストとエリアの紐付け

**User Story:** As a コンテスト主催者, I want コンテスト作成時に対象エリアを選択できる, so that コンテストの地域範囲を明確にできる

#### Acceptance Criteria

1. WHEN 主催者がコンテスト作成・編集画面を開く THEN システムは主催者のエリア一覧をドロップダウンで表示する
2. WHEN 主催者がエリアを選択 THEN システムは選択エリアの地図プレビューを表示する
3. IF エリアが未選択 THEN システムはエリアなしでコンテストを作成可能（全国対象等）
4. WHEN コンテストにエリアが設定されている THEN システムはコンテスト詳細画面にエリア情報と地図を表示する

### REQ-7: 投稿時のスポット選択

**User Story:** As a 参加者, I want 投稿時に撮影スポットを選択できる, so that どこで撮影したか記録できる

#### Acceptance Criteria

1. WHEN 参加者がスポット設定ありのコンテストに投稿 THEN システムはスポット選択UIを表示する
2. WHEN 参加者がスポットをドロップダウンから選択 THEN システムは選択スポットを投稿に紐付ける
3. WHEN 参加者が地図上のマーカーをクリックしてスポットを選択 THEN システムは選択スポットを投稿に紐付ける
4. IF コンテストのスポット選択が必須設定 AND スポット未選択 THEN システムは「撮影スポットを選択してください」エラーを表示する
5. IF コンテストのスポット選択が任意設定 THEN システムはスポット未選択でも投稿を許可する

### REQ-8: 地図表示（コンテスト詳細）

**User Story:** As a 参加者, I want コンテスト詳細画面で対象エリアとスポットを地図で確認できる, so that どこで撮影すればよいか把握できる

#### Acceptance Criteria

1. WHEN 参加者がエリア設定ありのコンテスト詳細を閲覧 THEN システムは Leaflet 地図にエリアを表示する
2. IF エリアに境界線が設定されている THEN システムはポリゴンで境界線を描画する
3. IF エリアに代表座標のみ設定 THEN システムは代表座標を中心にマーカーを表示する
4. WHEN コンテストにスポットが登録されている THEN システムはスポットをマーカーで表示する
5. WHEN 参加者がスポットマーカーをクリック THEN システムはスポット名とカテゴリをポップアップ表示する

### REQ-9: 地図表示（投稿一覧）

**User Story:** As a 参加者, I want 投稿一覧を地図上で閲覧できる, so that どのスポットでどんな写真が撮られているか把握できる

#### Acceptance Criteria

1. WHEN 参加者がコンテストの投稿一覧で地図モードを選択 THEN システムは投稿を地図上にマーカー表示する
2. WHEN スポットが設定された投稿 THEN システムはスポット位置にマーカーを表示する
3. WHEN 同一スポットに複数投稿がある THEN システムはクラスタリングして件数を表示する
4. WHEN 参加者がマーカーをクリック THEN システムは投稿のサムネイルとタイトルをポップアップ表示する
5. WHEN 参加者がポップアップの投稿をクリック THEN システムは投稿詳細画面に遷移する

## Non-Functional Requirements

### Code Architecture and Modularity

- **Single Responsibility Principle**: 地図関連のロジックは専用の JavaScript コントローラー（Stimulus）に分離
- **Modular Design**: Leaflet 関連の共通処理は再利用可能なヘルパー/ユーティリティとして実装
- **Dependency Management**: Leaflet は importmap 経由で管理、プラグインは最小限に
- **Clear Interfaces**: Area / Spot モデルは他機能から独立して利用可能な設計

### Performance

- 地図の初期表示は 2 秒以内
- 100 スポットまでのマーカー表示はスムーズに動作
- 境界線データ（GeoJSON）は 1MB 以下に制限

### Security

- 座標データは主催者のみ編集可能
- 他者のエリア・スポットへのアクセスは禁止
- GeoJSON は sanitize して保存

### Reliability

- 地図 API（OpenStreetMap タイル）が利用不可時は住所テキストのみ表示
- 座標取得失敗時は手動入力で代替可能

### Usability

- 地図操作は直感的（ドラッグ、ズーム、クリック）
- モバイルでもタッチ操作で地図を利用可能
- 境界線描画は取り消し・やり直し可能
